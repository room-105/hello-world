# ソフトウェア工学 2026 講義まとめ

これまでのソフトウェア工学の授業を通して、開発プロセスや開発ツールの使い方、そしてモダンなリリースフローについて体系的に学びました。

## 開発プロセスとコード品質の管理
* **WBSによるプロジェクト管理**
  作業を細かく分解して具体的なスケジュールを立てることで、開発全体の進捗を把握する重要性を実感しました。
* **flake8による静的解析とエディタの活用**
  普段使用しているVS Codeの強力さを改めて感じるとともに、flake8を用いたエラー解決を通して、コードの品質を高い水準で自動的に保つための静的解析の便利さを体験しました。

## バージョン管理の基礎と共同開発
* **Git/GitHubを用いたバージョン管理**
  チュートリアルサイトなどを活用しながら、コミット（commit）、ブランチの作成（branch）、マージ（merge）といったバージョン管理における一連の基本操作を確実に理解しました。

## GitHub PagesによるCI/CDの実践
* **自動ビルドとデプロイの統合**
  GitHub Pagesの設定において、ソースをGitHub Actionsに切り替えることで、コードをプッシュ・マージした際に自動的にWebサイトが構築・公開されるCI/CD（継続的インテグレーション／継続的デリバリー）の仕組みを実践しました。
* **Jekyllによる静的サイトの作成**
  Jekyllを利用することで、プレーンなMarkdownファイルを自動でHTML化し、手軽にWeb上にポートフォリオやレクチャーノートを公開する手法を習得しました。これにより、開発者はコンテンツの執筆に集中できる効率的なワークフローを体験できました。
